## 1. 项目定位

这是一个历史形成的灵枢枢机（Stratix）源码 monorepo，当前同时承载：

- 灵枢枢机核心框架与 CLI
- database / redis / queue / tasks / ossp / was-v7 / devtools / testing 等生态插件
- 非 workspace 的“幻廊之镜”CLI 预览样例 `examples/web-admin-preview`

本轮工作不是重写项目，而是把已有代码、配置、发布面和验证事实接入软件工厂治理。

## 2. 本轮目标

- 基于真实代码与命令结果建立当前状态基线
- 建立 `docs/`、`.factory/`、`AGENTS.md`、`GEMINI.md`
- 明确当前建议阶段、首批 `BUG/CR/TASK` 和后续推进顺序
- 把顶层入口文档中的瞬时状态迁移到 discovery / memory 层

## 3. 范围

### In

- 根仓库治理与文档体系
- monorepo 工作区结构、脚本、发布面、运行入口核验
- 首批问题与任务建档

### Out

- 一次性修复全部构建与测试问题
- 本轮内重做发布流程
- 为每个生态包补完整对外用户文档

## 4. 成功判定

- 当前真实状态可由人类和代理快速读取
- 稳定协作规则与瞬时状态完成分层
- 当前阶段及其阻塞项明确
- 后续修复工作可以按 `BUG/CR/TASK` 直接推进
