# 课程表同步系统UI设计方案

## 1. 设计原则

### 1.1 设计理念
- **简洁实用**：界面简洁明了，突出核心功能
- **一致性**：保持视觉和交互的一致性
- **响应式**：适配各种设备和屏幕尺寸
- **直观易用**：操作流程简单明确，降低学习成本
- **专注内容**：减少干扰，让用户专注于核心任务

### 1.2 设计规范
- **色彩系统**：
  - 主色：#4F46E5（靛蓝色）
  - 辅助色：#10B981（绿色）、#F59E0B（橙色）、#EF4444（红色）
  - 中性色：#000000、#6B7280、#E5E7EB、#FFFFFF
- **字体**：
  - 主字体：系统默认字体（无衬线）
  - 标题：16-24px，粗体
  - 正文：14px，常规
  - 辅助文本：12px，常规
- **布局**：
  - 网格系统：12列网格
  - 主内容区：最大宽度1200px，居中
  - 间距系统：4px的倍数（4px, 8px, 16px, 24px, 32px等）
  - 响应式断点：移动端（<640px）、平板（640px-1024px）、桌面（>1024px）

## 2. 公共组件

### 2.1 导航栏
- **设计特点**：固定在顶部，包含系统logo、主导航菜单、用户信息和操作按钮
- **组件构成**：
  - Logo：系统标识
  - 主导航：根据用户角色显示不同的菜单项
  - 用户信息：用户头像和姓名
  - 通知图标：显示未读通知数量
  - 设置图标：进入个人设置
- **交互行为**：
  - 下拉菜单：鼠标悬停在用户头像上显示下拉菜单
  - 响应式：在移动端转变为汉堡菜单

### 2.2 日历组件
- **设计特点**：显示课程安排，支持多种视图模式
- **组件构成**：
  - 视图切换：日视图、周视图、月视图
  - 时间导航：上一个/下一个/今天
  - 日历格子：显示日期和课程信息
  - 课程卡片：显示课程名称、时间、地点等
- **交互行为**：
  - 点击课程：显示课程详情浮层
  - 滑动操作：支持左右滑动切换日期
  - 缩放操作：支持双指缩放调整视图

### 2.3 统计卡片
- **设计特点**：简洁直观地展示统计数据
- **组件构成**：
  - 卡片标题：统计指标名称
  - 主数据：大号显示核心数据
  - 趋势指标：与上期相比的变化
  - 图表：简单的趋势线或饼图
- **交互行为**：
  - 悬停状态：显示详细说明
  - 点击操作：跳转到详细统计页面

### 2.4 表单组件
- **设计特点**：清晰易用的表单设计
- **组件构成**：
  - 输入框：带有标签和验证提示
  - 选择器：下拉选择、日期选择等
  - 按钮：主要按钮、次要按钮
  - 提示信息：错误提示、帮助文本
- **交互行为**：
  - 焦点状态：输入框获得焦点时的视觉反馈
  - 验证反馈：实时显示输入验证结果
  - 提交操作：提交后的加载状态和结果反馈

## 3. 学生端界面

### 3.1 学生首页
- **页面功能**：学生登录后的首页，展示个人课表和出勤概况
- **界面布局**：
  - 顶部：导航栏
  - 左侧：统计卡片（出勤率、待上课程、待打卡课程）
  - 右侧：今日课程列表
  - 主区域：课程日历（周视图）
- **关键元素**：
  - 出勤率环形进度条
  - 近期课程时间线
  - 快捷打卡入口
- **交互详情**：
  - 点击课程卡片：进入课程详情
  - 点击打卡按钮：直接进入打卡页面
  - 下拉刷新：更新课程和出勤数据

### 3.2 课程详情页
- **页面功能**：显示特定课程的详细信息
- **界面布局**：
  - 顶部：课程基本信息（名称、教师、时间等）
  - 中部：课程内容和要求
  - 底部：功能按钮（打卡、请假）
- **关键元素**：
  - 课程信息卡片
  - 授课教师信息
  - 上课地点地图
  - 考勤记录列表
- **交互详情**：
  - 地图交互：显示教学楼位置和导航
  - 教师卡片：点击查看教师详情
  - 考勤记录：滚动查看历史记录

### 3.3 打卡页面
- **页面功能**：学生进行课程打卡
- **界面布局**：
  - 顶部：课程信息和倒计时
  - 中部：地图显示当前位置和目标位置
  - 底部：打卡按钮和状态提示
- **关键元素**：
  - 打卡状态指示器（可打卡、已打卡、距离过远等）
  - 位置准确度指示器
  - 大型打卡按钮
  - 问题反馈入口
- **交互详情**：
  - 按钮状态：根据打卡条件自动变化
  - 打卡动画：成功打卡后的视觉反馈
  - 位置更新：实时更新位置信息
  - 错误处理：显示打卡失败原因和解决建议

### 3.4 请假申请页
- **页面功能**：学生提交请假申请
- **界面布局**：
  - 顶部：课程信息
  - 中部：请假表单
  - 底部：提交按钮和历史请假记录
- **关键元素**：
  - 请假类型选择器
  - 请假原因输入框
  - 附件上传区域
  - 审批流程预览
- **交互详情**：
  - 表单验证：实时验证输入内容
  - 预览功能：预览请假申请内容
  - 提交确认：二次确认提交
  - 状态跟踪：提交后自动进入状态跟踪

### 3.5 个人出勤统计页
- **页面功能**：展示学生个人出勤数据和统计
- **界面布局**：
  - 顶部：总体出勤率和排名
  - 中部：各课程出勤率对比图表
  - 底部：详细出勤记录列表
- **关键元素**：
  - 出勤率仪表盘
  - 课程出勤率柱状图
  - 出勤趋势线图
  - 筛选器和导出按钮
- **交互详情**：
  - 图表交互：悬停显示详细数据
  - 筛选操作：按课程、时间筛选数据
  - 排序功能：多维度排序出勤记录
  - 导出功能：导出数据到Excel

## 4. 教师端界面

### 4.1 教师首页
- **页面功能**：教师登录后的首页，展示教学日历和课程情况
- **界面布局**：
  - 顶部：导航栏
  - 左侧：课程列表和统计数据
  - 右侧：待办事项（待审批请假）
  - 主区域：教学日历
- **关键元素**：
  - 课程出勤率概览
  - 待审批请假计数
  - 今日教学提醒
  - 班级列表快捷入口
- **交互详情**：
  - 点击课程：进入课程详情
  - 点击待审批：进入请假审批页面
  - 日历交互：查看不同日期的课程安排

### 4.2 课程管理页
- **页面功能**：管理特定课程的学生和出勤情况
- **界面布局**：
  - 顶部：课程信息和操作按钮
  - 中部：班级出勤率统计
  - 底部：学生列表和出勤详情
- **关键元素**：
  - 课程信息卡片
  - 出勤率统计图表
  - 学生名单和出勤状态
  - 异常出勤标记
- **交互详情**：
  - 学生筛选：按出勤状态筛选学生
  - 出勤修改：手动调整出勤记录
  - 批量操作：批量处理学生出勤
  - 数据导出：导出班级出勤数据

### 4.3 请假审批页
- **页面功能**：处理学生的请假申请
- **界面布局**：
  - 顶部：筛选和搜索
  - 中部：待审批请假列表
  - 底部：历史审批记录
- **关键元素**：
  - 请假类型标签
  - 请假时间线
  - 申请详情卡片
  - 审批按钮组
- **交互详情**：
  - 请假详情：展开/收起请假详情
  - 审批操作：同意/拒绝请假申请
  - 批量审批：选择多个申请进行批量审批
  - 评论功能：添加审批意见

### 4.4 出勤统计页
- **页面功能**：查看和分析课程出勤数据
- **界面布局**：
  - 顶部：课程选择器和时间范围选择
  - 中部：班级出勤率图表和数据表格
  - 底部：详细出勤记录和异常分析
- **关键元素**：
  - 多维度统计图表
  - 出勤趋势分析
  - 异常出勤警示
  - 数据导出工具
- **交互详情**：
  - 数据钻取：从总览下钻到详细数据
  - 图表交互：调整图表类型和维度
  - 异常分析：突出显示异常出勤模式
  - 报表生成：生成定制化出勤报表

  - 异常出勤警示

  - 数据筛选：多条件组合筛选
  - 数据导出：多格式数据导出
  - 报表生成：生成学校出勤报告

## 6. 移动端适配

### 6.1 移动端架构
- **设计策略**：移动优先的响应式设计
- **核心原则**：
  - 简化界面：减少非必要元素
  - 优化触摸：加大点击区域
  - 流畅体验：减少页面切换
  - 离线支持：关键功能离线可用

### 6.2 学生移动端
- **重点功能**：
  - 快速打卡：优化打卡流程
  - 课程通知：课程提醒和变更通知
  - 请假申请：简化请假流程
  - 出勤查询：随时查看出勤状态
- **交互优化**：
  - 手势操作：支持滑动和手势操作
  - 推送通知：课程开始前推送提醒
  - 地图集成：提供教室位置导航

### 6.3 教师移动端
- **重点功能**：
  - 请假审批：快速处理请假申请
  - 课程提醒：上课提醒和课程变更通知
  - 简易点名：课堂快速点名工具
  - 出勤查看：随时查看学生出勤情况
- **交互优化**：
  - 批量操作：优化批量审批流程
  - 离线审批：支持离线审批后同步
  - 语音输入：支持语音输入审批意见

## 7. 辅助功能设计

### 7.1 无障碍设计
- **视觉辅助**：
  - 高对比度模式
  - 可调整字体大小
  - 屏幕阅读器支持
  - 键盘导航支持
- **交互辅助**：
  - 简化操作流程
  - 足够的操作响应时间
  - 明确的错误提示
  - 多种反馈方式（视觉、听觉）

### 7.2 国际化支持
- **多语言支持**：
  - 界面文本国际化
  - 日期时间格式本地化
  - 文化适应性考量
- **实现方式**：
  - 文本外部化
  - 动态加载语言包
  - 方向自适应（支持RTL布局）
