## 产品理念
一个Node.js应用框架，名称为stratix，以函数式编程思想为核心，追求简洁、透明和组合性。在stratix中，一切都是插件，routes、utilities、orm、cache、logger、queue、cron、cli、deploy、ops等都是插件，插件之间可以相互组合，形成一个完整的应用。


## 项目整体架构
整个项目和生态基于monorepo的方式进行开发，全部放在一个目录下每个包都是一个独立的项目，每个包都有自己的版本管理和发布流程。每个包都有自己的文档和示例，方便开发者学习和使用。项目整体说明放在根目录下的README.md文件中。
每个项目安装如下的方式组织代码：
```
├── docs
│   └── 设计文档.md
│   └── 示例.md
├── src
│── README.md
│── package.json
│── tsconfig.json
```
- 生态插件包位于packages目录下，以库的形式提供。
- 工程项目位于apps目录下，以应用的形式提供。可以打包成应用单独部署


## 框架说明
框架本身stratix 是一个极简的应用框架，所有的功能都通过插件Plugins实现,插件的功能参考fastify[https://github.com/fastify/fastify]库插件的功能实现，支持装饰器、封装、hooks等特性，以及统一的错误处理；框架基于awilix库[https://github.com/jeffijoe/awilix]提供Dependency Injection (DI) 能力

```typescript
import stratix from 'stratix';

const app = stratix();
app.register(myPlugin,{ options });
app.register(webPlugin,{ options });
...
app.start();
```
## 依赖
插件可以依赖其他插件，但是基于松耦合高聚合的方式开发。每个插件都不需要显示引用其他插件

## 插件定义
插件的定义和开发逻辑可以参照[https://fastify.dev/docs/latest/Guides/Plugins-Guide/]的内容，支持异步，要支持typescript定义，使用typescript开发插件

基础包和插件包的位于packages目录下

@stratix/cron 插件-定时任务包（基于node-cron 3.0.3实现）。项目路径packages/cron
@stratix/cli 插件-cli工具包（基于yargs 17.7.2实现）。项目路径packages/cli
@stratix/deploy 插件-部署包（基于pm2 6.0.5实现）。项目路径packages/deploy
@stratix/ops 插件-运维包（基于node-ops 1.0.0实现）。项目路径packages/ops

### 功能性插件
基于stratix框架及其生态开发的一个通讯录同步的项目，针对很多客户，不同的客户采集数据源的方式不一样，但是写入到目标系统的方式是一致的，单个项目的完整功能包括：
- 数据采集：从多个数据源（如LDAP、数据库、api接口、文件）采集通讯录数据。
- 数据清洗：对采集到的数据进行清洗和转换，确保数据的一致性和完整性。
- 数据写入：将清洗后的数据写入到目标系统（如api写入）。
- 数据同步：定期同步数据，确保数据的实时性和一致性。
- 数据查询：提供API接口，支持查询通讯录数据。
- 数据导入：支持从外部文件导入通讯录数据。
- 数据导出：支持将通讯录数据导出到外部文件。
- 数据备份：支持数据备份和恢复。
- 数据监控：支持数据监控和告警。

### 功能设计
基于这种情况，针对不同的客户，大部分功能是可以复用的，只有采集的部分是需要定制性开发的。基于这种情况，将这个功能分为三个部分，基于同一个中间数据库进行同步。
采集模块从数据源采集数据写入到中间库
写入模块从中间库读取数据写入到目标系统
- 数据采集：每次新项目都需要定制化开发，但是大部分功能是复用的，只有读取数据源和和将数据转换为中间库所需格式的部分是需要定制的，其他比如写入中间库、数据清晰等都是可以复用的，通过一个插件库的形式完成
- 数据同步和写入：由于中间库是固定，写入到目标系统的方式也是固定的，因此这部分内容是不需要重复开发的。
- 数据查询：由于中间库是固定，查询的方式也是固定的，因此这部分内容是不需要重复开发的。
### 功能插件包
功能插件包的位于packages目录下
- @obsync/data-collector 插件-数据采集包的公共代码
- @obsync/data-distributor 插件-数据同步写入插件
另外数据同步写入时需要调用api完成目标系统的写入，需要实现调用api的插件。需要分钟v1和v7两个版本的插件包。
- @obsync/wapi-v1 插件-调用api的插件包
- @obsync/wapi-v7 插件-调用api的插件包
### 项目插件包
项目代码位于apps目录下
- @obsync/template 项目模板包，用于快速创建一个新的项目。实现采集模块中的数据源抓取，代码要尽可能的简单，尽量通过配置的方式实现，我已经写好了模版的代码，要根据我的代码反推实现。部署时部署成一个服务。
- @obsync/capsulex 实现了同步应用的web界面，通过界面管理数据写入部分，基于@obsync/data-distributor插件包实现。部署成单一的服务。
