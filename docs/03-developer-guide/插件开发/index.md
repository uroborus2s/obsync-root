# 插件开发概览

本目录面向零经验插件开发者和生态维护者，目标不是“知道有哪些插件”，而是让你看完后能独立做出一个可接入 Stratix 应用的生态插件。

## 建议阅读顺序

1. [快速开始](./getting-started.md)
2. [项目结构](./project-structure.md)
3. [第一个生态插件实战](./build-your-first-plugin.md)
4. [开发工作流](./development-workflow.md)
5. [插件目录](./plugin-catalog.md)
6. [组合指南](./composition-guide.md)

## 适用对象

- 生态插件维护者
- 需要扩展生态能力的开发者
- 需要评估插件组合策略的架构设计者

## 维护约束

- 本目录聚焦插件开发与生态扩展，不重复展开框架内部研发过程。
- 教程必须能让一个没有 Stratix 插件经验的人照着完成初始化、编码、接入和验证。
- 新插件进入公共生态时，要同步补齐目录、职责、组合影响和扩展边界说明。
