# 开发者指南概览

本目录面向实际开发者，包含一套按版本维护的函数 API 参考，以及三条固定主线：应用后端开发、脚手架开发、插件开发。

## 公共 API 入口

- [函数 API 版本索引](./function-api/index.md)：统一维护 `@stratix/core` 及其子路径的公共导出面文档。
- [函数 API v1.1.0 概览](./function-api/v1.1.0/index.md)：第一版正式参考入口，按 `core`、`service`、`functional` 三页组织。

## 三条主线

- `应用后端开发/`：面向零经验后端开发者，目标是看完后能独立完成项目初始化、接口开发、数据库接入和模块化演进。
- `脚手架开发/`：面向前端模板使用者，目标是独立生成、理解、扩展并验收幻廊之镜管理后台脚手架。
- `插件开发/`：面向零经验插件开发者和生态维护者，目标是看完后能独立初始化、实现、接入并验证一个生态插件。

## 适用对象

- 基于 `@stratix/cli` 创建和维护项目的开发者
- 需要在现有模板和生态上继续扩展能力的团队成员

## 维护约束

- 本目录只讨论“如何基于框架开发”，不重复展开框架内部研发过程。
- 三条主线的阅读顺序与页面组织必须保持稳定，避免新手失去路径感。
- 公共 API 页面需要与 `packages/core/package.json` 的 `exports` 和 `packages/core/src/index.ts` 保持同步。
- 函数 API 参考按版本维护，首个正式版本从 `v1.1.0` 开始。
