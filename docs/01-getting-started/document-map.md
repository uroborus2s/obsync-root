# 文档地图

本仓库文档按四个一级目录维护，但一级名称已经对齐 docs-stratego 标准结构。

## 1. 入门说明

`01-getting-started/` 用来回答“这个项目是什么、我该从哪里开始看”。

## 2. 用户指南

`02-user-guide/` 面向使用者、接手者和新维护者，解决“怎么理解当前仓库、怎么继续接手维护”。

## 3. 开发者指南

`03-developer-guide/` 面向实际开发者，并进一步拆成三部分：

- `应用后端开发/`：基于灵枢枢机构建后端业务应用
- `脚手架开发/`：基于幻廊之镜样板扩展前端管理后台
- `插件开发/`：理解生态插件边界、组合关系与扩展方式

## 4. 项目开发文档

`04-project-development/` 记录框架自身的治理、调研、需求、方案、交付、测试、发布、运维和追踪矩阵。

## 推荐阅读分流

- 新同学先看 [快速开始](./quick-start.md)
- 后端开发者去 `03-developer-guide/应用后端开发/`
- 前端脚手架开发者去 `03-developer-guide/脚手架开发/`
- 插件维护者去 `03-developer-guide/插件开发/`
- 框架维护者去 `04-project-development/`
