# 快速开始

如果你第一次进入这个仓库，建议按下面顺序阅读和行动：

1. 先看 [项目概览](./project-overview.md)，理解仓库边界和文档分层。
2. 再看 [文档地图](./document-map.md)，确认自己应该进入哪一条文档链路。
3. 如果你要做后端应用开发，进入 [开发者指南 / 应用后端开发](../03-developer-guide/应用后端开发/getting-started.md)。
4. 如果你要做前端脚手架开发，进入 [开发者指南 / 脚手架开发](../03-developer-guide/脚手架开发/getting-started.md)。
5. 如果你在维护框架本身，进入 [项目开发文档 / 当前状态分析](../04-project-development/02-discovery/current-state-analysis.md)。

## 常用入口

- 仓库说明：`README.md`
- 协作约束：`AGENTS.md`、`GEMINI.md`
- 当前状态：`../04-project-development/02-discovery/current-state-analysis.md`
- AI 状态压缩：`../.factory/project.json`、`../.factory/memory/current-state.md`

## 进入开发前的最低要求

- 使用仓库声明的 Node.js 与 `pnpm` 版本
- 不把历史 README 当成已验证事实，优先看当前状态文档
- 文档、代码、测试和 `.factory/` 需要同步维护
