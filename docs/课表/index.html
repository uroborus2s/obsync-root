课表同步系统 - UI设计文档

本文档展示了课表同步系统的界面设计，包括同步管理页面和详情模态窗口。该系统基于产品需求文档设计，旨在将教育机构课表数据智能转换为WPS协作应用中的各类应用数据。

同步管理页面 同步详情模态窗口 设计系统

1. 同步管理页面

同步管理页面是课表同步系统的核心功能页面，提供了同步策略配置、数据转换规则设置、同步状态监控和错误处理等功能。管理员可以在此页面管理所有与课表同步相关的配置和操作。

主要功能模块

  • 同步策略配置：设置同步频率（手动、每日、每小时、实时）、同步方式（全量/增量）、失败重试次数和同步时间段。
  • 数据转换规则：配置课表数据到日历、打卡和审批流的转换规则，包括格式模板、提醒设置和可见性控制。
  • 同步状态监控：实时展示同步任务的执行状态，包括任务ID、同步类型、数据源、目标应用、开始/结束时间和状态。
  • 错误处理：显示同步过程中发生的错误，提供错误详情查看和重试功能。

2. 同步详情模态窗口

同步详情模态窗口提供了单个同步任务的详细信息，包括基本信息、任务进度、错误信息、详细日志和统计数据。管理员可以通过此窗口深入了解同步任务的执行情况和问题原因。

详情模态窗口功能

  • 基本信息：展示任务ID、同步类型、数据源、目标应用、发起人、状态和时间等基本信息。
  • 任务进度：以时间线的形式展示任务执行的各个阶段和状态。
  • 错误信息：当任务失败时，提供详细的错误信息和解决建议。
  • 详细日志：展示任务执行过程中的详细日志，包括信息、警告和错误级别的日志。
  • 统计数据：提供任务执行的数据统计，如总记录数、成功/失败数量、耗时等。

设计规范

  • 一致性原则：所有页面采用统一的设计语言，确保用户体验的一致性。
  • 简洁明了：界面设计简洁明了，突出核心功能，减少视觉干扰。
  • 信息层次：清晰的信息层次，通过颜色、大小和间距等区分重要性。
  • 响应式设计：采用响应式设计，确保在不同屏幕尺寸下都有良好的展示效果。
  • 状态明确：通过醒目的视觉元素明确表示各种状态（成功、警告、错误等）。
