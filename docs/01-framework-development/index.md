# 灵枢枢机框架开发过程文档

本目录维护灵枢枢机（Stratix）框架与公共生态的治理、调研、需求、方案、交付、质量、发布、运维、交接和演进文档。

优先阅读：

1. `01-discovery/current-state-analysis.md`
2. `02-requirements/prd.md`
3. `03-solution/system-architecture.md`
4. `04-delivery/implementation-plan.md`
5. `05-quality/test-plan.md`

阶段目录：

- `00-governance/`: 项目治理、边界和项目章程
- `01-discovery/`: 输入、现状、事实验证和差距分析
- `02-requirements/`: PRD、需求分析和需求校验
- `03-solution/`: 架构、模块边界、技术选型和 API 设计
- `04-delivery/`: 实施计划和交付节奏
- `05-quality/`: 测试计划和质量控制
- `06-release/`: 发布说明
- `07-operations/`: 部署与运行维护
- `08-handover/`: 使用和交接材料
- `09-evolution/`: 后续演进文档
- `traceability/`: 需求追踪矩阵

说明：

- 当前验证结论以 `01-discovery/current-state-analysis.md` 与 `.factory/` 为准。
- 这一类文档主要服务于框架自身的研发、治理和发布过程，不直接替代应用开发者指南。
