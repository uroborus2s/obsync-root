# 课程维度签到统计前端功能测试指南

## 功能概述

课程维度签到统计页面提供按课程维度查看签到统计数据的功能，包括数据筛选、排序、分页和统计概览。

## 页面访问

**页面路径：** `/attendance/course-statistics`

**导航位置：** 签到管理 → 课程维度统计

## 功能测试

### 1. 页面加载测试

**测试步骤：**
1. 登录系统
2. 点击左侧导航"签到管理"
3. 点击"课程维度统计"菜单项
4. 验证页面正常加载

**预期结果：**
- 页面标题显示"课程维度签到统计"
- 筛选条件区域正常显示
- 统计概览卡片显示数据
- 数据表格正常加载

### 2. 筛选功能测试

**测试用例2.1：学期筛选**
1. 在"学期"输入框输入"2024-2025-1"
2. 验证数据自动刷新
3. 确认显示的数据符合学期条件

**测试用例2.2：课程名称筛选**
1. 在"课程名称"输入框输入"数学"
2. 验证数据自动刷新
3. 确认显示的课程名称包含"数学"关键词

**测试用例2.3：教师工号筛选**
1. 在"教师工号"输入框输入教师工号
2. 验证数据自动刷新
3. 确认显示的数据为该教师的课程

**测试用例2.4：组合筛选**
1. 同时设置多个筛选条件
2. 验证数据符合所有筛选条件

### 3. 排序功能测试

**测试用例3.1：按签到率排序**
1. 选择排序方式为"签到率"
2. 选择排序方向为"降序"
3. 验证数据按签到率从高到低排列

**测试用例3.2：按出勤率排序**
1. 选择排序方式为"出勤率"
2. 选择排序方向为"升序"
3. 验证数据按出勤率从低到高排列

**测试用例3.3：按学生数量排序**
1. 选择排序方式为"学生数量"
2. 验证数据按学生数量排序

**测试用例3.4：按上课时间排序**
1. 选择排序方式为"上课时间"
2. 验证数据按时间顺序排列

### 4. 分页功能测试

**测试用例4.1：分页导航**
1. 验证分页信息显示正确（当前页/总页数）
2. 点击"下一页"按钮，验证页面跳转
3. 点击"上一页"按钮，验证页面跳转
4. 验证首页和末页按钮状态正确

**测试用例4.2：分页数据一致性**
1. 记录第一页的数据
2. 切换到第二页
3. 返回第一页，验证数据一致

### 5. 数据展示测试

**测试用例5.1：表格数据完整性**
验证表格包含以下列：
- 课程名称
- 课次信息
- 周次
- 任课教师
- 学生数量
- 状态
- 签到数量
- 请假数量
- 缺勤数量
- 签到率
- 出勤率

**测试用例5.2：状态标签显示**
1. 验证"未开始"状态显示为灰色标签
2. 验证"进行中"状态显示为蓝色标签
3. 验证"已结束"状态显示为默认标签

**测试用例5.3：签到率颜色编码**
1. 验证签到率≥85%显示绿色
2. 验证签到率70-84%显示黄色
3. 验证签到率60-69%显示橙色
4. 验证签到率<60%显示红色

**测试用例5.4：出勤率颜色编码**
1. 验证出勤率≥90%显示绿色
2. 验证出勤率80-89%显示黄色
3. 验证出勤率70-79%显示橙色
4. 验证出勤率<70%显示红色

## 测试环境要求

- 浏览器：Chrome、Firefox、Safari、Edge最新版本
- 网络：稳定的网络连接
- 权限：教师或管理员账号
- 数据：测试环境需要有足够的测试数据

## 已知问题和限制

1. 大数据量时可能需要优化加载性能
2. 复杂筛选条件组合可能影响查询速度
3. 移动端表格滚动体验有待优化
